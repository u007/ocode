### Changed
- **Discovery Ignore Paths Merge Semantics** — Custom `discovery.ignore_paths` entries in `ocodeconfig.json` are now *merged with* the built-in default list rather than replacing it. Previously, setting `"ignore_paths": ["docs/"]` would ignore only `docs/`; now the built-in defaults (`skills/`, `.opencode/`, `.claude/`, etc.) are always included alongside user entries. The `/discover ignore clear` command resets to defaults rather than clearing the list entirely. If you relied on `ignore_paths: []` to disable all ignores, your config now receives the defaults.
- **Project Markdown Discovery Prompting** — `/discover` now keeps project `*.md` files out of the stable name-index entirely unless they are attached for the current turn. Attached docs are injected as filename + summary only; full file bodies are no longer pre-injected into the prompt.
- **Discovery Selection Policy** — Added `SelectMin` (0.40) as an absolute minimum similarity threshold; `SelectRankRelative` now requires items to pass BOTH the relative delta (`SelectDelta=0.15`) AND the absolute minimum (`>=SelectMin`). `SelectFloor` reduced from 5 to 0, eliminating forced low-quality skill attachments. Locked in by updated `TestSelectRankRelativeBounds`.
- **Shell Command Output Preservation in Session Restore** — `buildAgentMessagesSnapshot` now converts shell-`*` tool_call+result pairs into `Role:"user"` messages instead of stripping them, preserving `!command` shell output in the LLM history across session restores. This avoids invalid DeepSeek message ordering (synthesized assistant tool_calls before first user message) and prevents orphaned tool results from contaminating the message stream. Locked in by `TestBuildAgentMessagesSnapshotKeepsShellOutputSeparate`.
- **`/context` Discovery-Aware Labels** — `/context` now shows `"Skill catalog (not pre-injected — discovery active)"` when discovery is enabled, and splits attached skills from MCP tools in the Discovery section. Context budget excludes skill catalog tokens when discovery is active, reflecting the net savings from gating.
- **`/discover status` Split Output** — `/discover status` now displays attached skills and MCP tools as separate lists with per-category counts, making it easier to see what's actually attached.
- **Single-width ASCII Status Symbols** — Replaced all double-width emoji used as inline status prefixes (`📄`, `🔧`, `📕`, `✨`, `🌐`, `⚡`, `🧠`, `📦`, `✅`, `📋`, `🔗`, `📎`, `🗑️`, `💬`, `🤖`, `🔖`) with single-width ASCII symbols (`▸`, `⚙`, `✦`, `⊕`, `›`, `◆`, `▣`, `✓`, `≡`, `~`, `+`, `∅`, `»`, `@`, `§`) across `kindIcon()`, CLI summary output (`summary.go`), LSP format helpers (`lsp_format.go`), git model status messages, TUI hints/notices, and the agent detail view. ### Fixed
- **Discovery Corpus Warm Synchronous + Timeout** — `RunDiscovery` now calls `Warm` synchronously with a 500ms deadline instead of spawning an async background goroutine. This ensures skills are ranked and attached on the very first turn. The timeout bounds first-turn latency on cold embed cache: if warm times out, the call falls through to the allow-all state (avoiding zero-tool attachments) and retries on the next turn. Removed `WarmAsync`, `warmOnce`, and `warming` from `Engine` (dead code after sync migration). Updated `discoveryAllows` comment to clarify that `Ready() == false` now signals a warm failure, not in-progress warming.
- **Shell Command Queuing During Streaming** — `!shell` commands entered while the agent is streaming or compacting are now queued and drained in `drainQueuedCommands` after the stream completes. Extracted the shell-execution setup logic into a reusable `startShellExecution` helper. Prevents loss of user input during concurrent streaming.
- **Bundled Skills FS Registration in `serve` and `web`** — The `serve` and `web` subcommands now register the embedded skills FS and model config FS, matching the registration already present in `skills`, TUI, and other subcommands. Fixes missing skill discovery in the `/api/skills` endpoint and discovery ranking when using the web UI or `serve` subcommand.
