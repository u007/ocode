# Plan: Add Novita live model fetch with pricing and context

**Created:** 2026-07-07
**Status:** Draft

## Context
The Novita AI provider is registered in auth and client config, but models come only from the static models.dev snapshot. Users want to see Novita's live model list (with pricing and context size) when they have credentials configured, similar to LM Studio and Requesty live fetches.

## Goals
1. Fetch live model list from Novita's API when credentials are available
2. Include context window size and pricing data from the API response
3. Fall back to the models.dev snapshot if the live fetch fails
4. Ensure `ModelCost` can resolve pricing for live-fetched models

## Approach
Follow the established LM Studio/Requesty pattern:
1. Add `fetchNovitaLiveModels()` — calls `GET /v1/models` on Novita's base URL, parses the rich response including `context_size`, `input_token_price_per_m`, `output_token_price_per_m`, features
2. Add a package-level cache (`novitaLiveData`) for the fetched model entries so `ModelCost` can access pricing
3. Skip `novita-ai` in snapshot-based model listing; use live fetch with snapshot fallback
4. Handle `novita-ai` in `providerModelsFromRegistry()` for individual provider lookups
5. Modify `modelEntryFor()` to check the Novita live cache as an additional data source

## Files to Change
- `internal/agent/models_registry.go` — add live fetch, cache, and integrate into all model listing/lookup paths

## Implementation Steps
1. Define response types and add `fetchNovitaLiveModels()` with API call
2. Skip novita-ai in `allProviderModelsFromRegistry()`, add live models with snapshot fallback
3. Handle novita-ai in `providerModelsFromRegistry()` 
4. Check novita live cache in `modelEntryFor()` so `ModelCost` resolves pricing

## Risks & Considerations
- Pricing unit conversion: Novita API returns `input_token_price_per_m` in 1/10000th of a dollar (divide by 10000 to get USD/M)
- API call must only fire when credentials are available (env var or stored key)
- Must handle API errors gracefully and fall back to snapshot
