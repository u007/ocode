# Plan: Agent view purge + token usage display

**Created:** 2026-07-06 14:05:10 +08
**Status:** Draft

## Context
The agent view (agent strip + detail) shows all sub-agent/background runs accumulated over a session. Completed runs pile up forever, making the strip scroll-heavy. Token usage is tracked per run (`AgentRun.inputTokens/outputTokens`) and for the main LLM (`sessionTelemetry`), but not surfaced in the collapsed agent strip or persistently on the chat tab status bar.

## Goals
1. Auto-purge completed agent runs so the list doesn't grow unbounded
2. Show input/output token usage via symbols on:
   - The chat tab status line (main LLM session total)
   - The agent strip (per-agent collapsed card)
   - The agent detail card header (per-agent, supplementing the existing status bar)

## Approach
1. Add `PruneCompleted` to `AgentRunRegistry` — removes completed runs beyond a `keepMax` threshold (keep 30 max, sorted by recency)
2. Call prune from `Snapshot()` so it auto-trims before TUI reads
3. Add token display to `renderStatus()` (chat tab status line)
4. Add token display to `renderAgentStrip()` (collapsed agent card header)
5. Add token display to `renderAgentRunCard()` (detail card header)

## Files to Change
- `internal/agent/agent_runs.go` — add PruneCompleted method
- `internal/tui/model.go` — add token usage to renderStatus() and renderAgentStrip()
- `internal/tui/detail_view.go` — add token usage to renderAgentRunCard() header

## Implementation Steps
1. Add `PruneCompleted(keepMax int)` to AgentRunRegistry, call from Snapshot()
2. In renderStatus(): show `↓Σinput ↑Σoutput` after LLM state when sessionTelemetry has data
3. In renderAgentStrip(): show `↓input ↑output` per-run after model label
4. In renderAgentRunCard(): show `↓input ↑output` in the card header

## Risks & Considerations
- `PruneCompleted` modifies the registry from `Snapshot()` — benign since the prune is lightweight (≤50 items) and read-safe (mutex-held)
- Token symbols must be single-width: use `↓` (U+2193) and `↑` (U+2191)
- Existing `formatTokenCount` (in model.go) handles k/M formatting — reuse it
- main LLM token display shows session-wide aggregate (includes side-channel calls)
